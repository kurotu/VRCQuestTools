# 変換後のアバターを確認する

マテリアル変換では、PC 用シェーダーの見た目を完全には再現できません。
また、非対応コンポーネントの削除により、ギミックの動作が変わることもあります。
どこがどう変わったのかは、実際に表示して確かめるのが確実です。
Mobile 用の実機がなくても、変換結果は PC 上で確認できます。

## PC 上で見た目を確認する

NDMF を使う場合は、次の 2 つの方法を使えます。

- **シーンでプレビューする**：[VQT Avatar Converter Settings](../components/avatar-converter-settings.md) のインスペクターで「[NDMF] 一時的なプレビューを有効化」ボタンを押します。変換後のマテリアルが Scene ビューに表示されます。もう一度押すと元の表示に戻ります。
- **VRChat のローカルテストで確認する**：Hierarchy でアバターを右クリックし、「VRCQuestTools」→「[NDMF] Build and Test for PC with Mobile Settings」を選択します。Mobile 用の設定で変換したアバターを、PC の VRChat でローカルテストできます。

手動変換の場合は、シーンに作成された「(アバター名) (Android)」をそのまま確認できます。
VRChat SDK のローカルテスト（Build & Test）を使うと、アップロードせずに VRChat 内で見た目を確認できます。

## 確認しておきたいポイント

- **色と明るさ**：Mobile 用シェーダーはライティングの仕組みが異なるため、ワールドによって明るさの印象が変わります。
- **透過表現**：Mobile 用シェーダーではテクスチャの透過が反映されません。頬染めやメガネのレンズなど、透過に頼っていた表現ほど影響が出やすい箇所です。対策は[トラブルシューティング](../troubleshooting.md#transparency)を参照してください。
- **削除されたコンポーネントの影響**：非対応コンポーネントの削除により、ギミックが動かなくなっていないか確認してください。
- **PhysBone の動作**：削減した PhysBone の揺れ方に問題がないか確認してください。

## Mobile 実機で確認する

アップロード後は、Quest などの実機でも表示を確認してください。
パフォーマンスランクが Very Poor の場合、他のプレイヤーからはデフォルトでインポスターまたはフォールバックアバターとして表示されます。
自分の画面で見えている姿と、相手に見えている姿が同じとは限りません。
