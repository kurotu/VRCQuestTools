# Convert Your Avatar

Add the converter settings to your avatar and make it ready to upload for Mobile.

:::caution Upload the PC version first
To synchronize PhysBone swaying between PC and Mobile, the same avatar must be uploaded to both platforms.
If you haven't uploaded the original avatar for PC yet, upload it first.
:::

## Begin Converter Settings

1. Select "Tools" → "VRCQuestTools" → "Setup Avatar for Mobile" from the menu bar.
2. Select your avatar in the window.
3. Press the "Begin Converter Settings" button.

![Setup Avatar for Mobile window](/img/convert-avatar.png)

The [VQT Avatar Converter Settings](../components/avatar-converter-settings.md) component is added to the avatar and the converter settings appear.
When the project has Modular Avatar or Avatar Optimizer, you can also choose additional components to add with the checkboxes above the button.

You can convert the avatar with the default settings.
If you want to change the conversion method per material or adjust texture resolutions, see the settings of [VQT Avatar Converter Settings](../components/avatar-converter-settings.md).

## When Warnings Are Shown

Depending on the avatar, warnings appear in the settings.
Typical ones are:

- **Removal of unsupported components**: Components which can't be used on Mobile (such as Constraints) are removed during conversion. After the conversion, check that the avatar's features still work.
- **Dynamic Bone**: VRCQuestTools does not convert Dynamic Bone to PhysBone. Convert them to PhysBones with the VRChat SDK feature beforehand.
- **Avatar Dynamics performance**: When there are too many PhysBones or Contacts, the components are removed even if the avatar can be uploaded. You need to [reduce PhysBones](../troubleshooting.md#avatar-dynamics).

## Upload

The following steps depend on whether the project has NDMF.

### With NDMF (Recommended)

The avatar is automatically converted on upload.
Because the original avatar is not modified, you don't need separate scenes or avatars for PC and Mobile.

1. Open the VRChat SDK control panel and select the "Builder" tab.
2. Switch the platform to Android in "Selected Platform(s)".
3. Upload the avatar with "Build & Publish" as usual.

:::info Screenshot placeholder
A screenshot of switching the platform in the VRChat SDK control panel will be placed here.
:::

To upload for the iOS version of VRChat as well, also select iOS in "Selected Platform(s)".
Switching the platform to iOS requires [iOS Build Support](./set-up-environment.md#ios-build-support).

### Without NDMF

Create a converted copy of the avatar with "Manual Conversion", then upload it.

1. Press the "Convert" button in the converter settings.
2. After the conversion, a copy named "(avatar name) (Android)" is created in the scene. The original avatar is automatically deactivated.
3. Switch the platform to Android (or iOS for the iOS version) in the VRChat SDK control panel.
4. Upload the converted copy with the same Blueprint ID as the original avatar.

Materials and textures generated by the conversion are saved in the folder shown as "Folder to Save".
